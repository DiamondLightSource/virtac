Using Virtac#

Before reading this document, ensure that Virtac has been either been installed using pip or you are able to run the docker image.

Setting your CA port#

If you are running the Virtac within diamond, you will need to change your CA ports so that the PVs created by the Virtac do not clash with the PVs for the real machine.

If you pip installed Virtac, this can be done by running the commands in your terminal:

$ export EPICS_CA_SERVER_PORT=8064
$ export EPICS_CA_REPEATER_PORT=8065

The docker image uses these ports by default so you can skip this step. But if you wish to change them to something else, you can pass the following arguments to your docker run command:

-e EPICS_CA_SERVER_PORT=8064 -e EPICS_CA_REPEATER_PORT=8065

Running Virtac#

For Virtac itself:

usage: virtac [-h] [-e] [-t] [-v] [--version] [ring_mode]

positional arguments:
  ring_mode             The ring mode to be used, e.g., IO4 or DIAD

options:
  -h, --help            show this help message and exit
  -e, --disable-emittance
                        Disable the simulator's time-consuming emittance
                        calculation
  -t, --disable-tfb     Disable extra simulated hardware required by the Tune
                        Feedback system
  -v, --verbose         Increase logging verbosity. Default is WARNING.
                        -v=INFO -vv=DEBUG
  --version             show program's version number and exit

If you have installed Virtac with pip, then you should be able to activate your virtual environment and run virtac with:

virtac

If you wish to use the container, then ensure that you have a working podman or docker install and then run virtac with:

podman run -it --network=host virtac:latest

Interactive shell commands#

Most users will not need to touch the interactive shell, and now know all they need to start using the Virtac and reading/writing to it using the PV interface.

But for completeness, there are is some additional functionality that you can access from the python interactive shell:

  • dbl() which lists all Virtac PVs

  • server.print_virtac_stats() which prints information about the Virtac, you can pass in an integer verbosity value with bigger numbers printing more information.

  • server.disable_monitoring() and server.enable_monitoring() which allow you to pause/resume the virtac.
